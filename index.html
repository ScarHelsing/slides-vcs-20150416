VCS

Version Control System - програмне забезпечення для полегшення роботи з інформацією, яка змінюється. Система керування версіями дозволяє зберігати кілька версій одного і того ж документа, при необхідності повертатися до більш ранніх версій, визначати, хто і коли зробив ту чи іншу зміну, і багато іншого.

Subversion — централізована система. Дані зберігаються в єдиному сховищі. При збереженні нових версій використовується дельта-компресія, тобто система знаходить відмінності нової версії від попередньої і записує тільки їх, уникаючи непотрібного дублювання даних

Git — розподілена система керування версіями файлів та спільної роботи. Git є однією з найефективніших, надійних і високопродуктивних систем керування версіями, що надає гнучкі засоби нелінійної розробки, що базуються на відгалуженні і злитті гілок.

Versioning

Commit - фіксує нові зміни у файлах, іменується кожен commit хешем sha1

.gitignore

Файл описує маски файлів, які не повинні попадати в commit, іншими словами, ці файли ігноруються

			.env
			/node_modules
			/static/content/
			/cache/*

Status

Відображає поточний стан репозиторію

$ git status

Pull

"Потягнути" дані з сервера в локальний репозиторій

$ git pull

Branching

Розбивання на гілки дозволяє паралельно працювати над різним функціоналаом, не заважаючи іншим

$ git checkout -b develop

Merge

Обєднання змін з різник гілок 
			$ git checkout master
			$ git merge develop

Diff

Візуальне відображення змін

Conflict
